# Deterministic-Prompt-Protocol
DPP, 결정론적 프롬프트 프로토콜, 5단계 프롬프트 구조

1. 역할정의: (AI를 빙의 시키는 것. 프롬프트가 충분하게 촘촘히 짜여져 있다면 사실 필요 없을 수도 있음. 단, 전체 프롬프트 구조가 엉성하거나 확실한 방향을 정해줘야 할 때는 꽤 유용함) 
2. 작업목표: (작업의 목적과 목표설정. 작업을 진행하는 매 순간 엉뚱한 방향으로 흘러가는 것을 방지해 줌. 바다위의 등대 같은 역할) 
3. 실행규칙: (일반적으로 말하는 프롬프트가 여기에 해당됨. 작업의 흐름, CoT를 결정해 줌)
4. 제약사항: (~하지마라. 또는 고려사항. 조심해서 사용해야 함) 
5. 출력프로토콜: (가장 중요함. 출력형식을 지정해 주는 것. AI가 딴 생각 못하도록 강제시켜 주는 가장 강력한 주문)

이 중 3. 실행규칙과 5. 출력프로토콜을 적절히 이용하여 CoT 를 결정해 주는 것이 중요함. 
이를 통해 AI의 할루시네이션을 통제함. 
할루시네이션을 제어한다는 의미는 창의성을 제한한다는 의미와 동일함
